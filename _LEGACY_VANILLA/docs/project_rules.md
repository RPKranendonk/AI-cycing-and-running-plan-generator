# Project Rules & Guidelines

## 1. Time Formatting
- **Rule:** ALWAYS use "morning" and "evening" to denote time of day.
- **Forbidden:** Do NOT use "AM" or "PM".
- **Reason:** Consistency and user preference.

## 2. Intensity Formatting
- **Rule:** ALWAYS use "% of Pace" for running intensity (e.g., "65-75% Pace").
- **Forbidden:** Do NOT use raw Zone names (e.g., "Z2" or "Zone 2") in final workout descriptions pushed to Intervals.icu.
- **Reason:** Intervals.icu interprets zones differently based on user settings; % of Pace is unambiguous.

## 3. Workout Structure
- **Rule:** Intervals must be defined as nested steps (Work/Rest) inside a Repeater step to ensure correct formatting.
- **Forbidden:** Do not merge intervals into a single text block.

## 4. Zone Validation (Internal)
- **Rule:** When displaying weekly stats, aggregate time into "Scientific Zones":
    - **Zone 1 (Low):** Friel Z1 + Z2
    - **Zone 2 (Moderate):** Friel Z3 + Z4
    - **Zone 3 (High):** Friel Z5a + Z5b + Z5c
